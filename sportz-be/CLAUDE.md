- **WebSocket**: `ws` library, `noServer: true`

- **Security**: ArcJet (rate limiting + bot detection)

## Key Architecture Rules

1. **REST and WebSocket share one port** вЂ” `http.createServer(app)` + `noServer: true`
2. **ArcJet must intercept `upgrade` event** вЂ” not `ws.on('connection')`. Use `noServer: true`.
3. **Broadcast AFTER DB write** вЂ” never inside a transaction
4. **Deduplication is mandatory** вЂ” track `lastSeenBall` per match using `overSep.balls` from Cricbuzz. Same ball must never broadcast twice.
5. **`ws.terminate()` not `ws.close()`** вЂ” for forced cleanup of ghost connections
6. **Check `ws.readyState` and `ws.bufferedAmount`** before every send
7. **`sequence` is the reconnect cursor** вЂ” not `createdAt`. Composite index `(matchId, sequence)` is the hot path.
8. **`RAPIDAPI_KEY` is server-side only** вЂ” never referenced in any client-facing code
9. **Neon pooler endpoint** вЂ” not the direct endpoint. Append `?sslmode=require`.
10. **Polling only runs for `live` matches** вЂ” check `status = 'live'` before starting a poller

## Cricket-Specific Notes

- Match formats: T20, ODI, TEST
- Innings periods: `1ST_INN`, `2ND_INN`, `SUPER_OVER`
- Over notation: `"15.4"` = over 15, ball 4
- Tournaments covered (free): IPL, ICC World Cup, bilateral series
- No live matches in off-season вЂ” use seed script for development
